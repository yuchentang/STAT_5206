What happens when we fit the wrong model to the data?¶

It is common in Statistics to wonder the impact of fitting the wrong model to the data.

For this exercise, we will focus on fitting unimodal distributions to multi-modal data.

Problem 0, simulating multi-modal data via a Gaussian Mixture¶

You can imagine a distribution with $k$ modes comes from overlaying $k$ different Gaussian distributions on top of each other. To simulate something like that, we could define:

$$X \sim MultiNomial(p_1, p_2, \dots, p_k)$$$$Y|X \sim Normal(\mu_{X}, \sigma^2_{X})$$

where $p_i = P(X=i)$, then we say $Y$ is a mixture Gaussian with $k$ mixtures. There will be $3k$ number of parameters overall, $p_1, \dots, p_k$, $\mu_1, \dots, \mu_k$, and $\sigma^2_1, \dots, \sigma^2_k$.

The 2 stage expression above suggests how we could simulate this using standard distributions like the multinomial and Gaussian.

  • Before you start coding, write down in words, how you would translate the mathematical expression into a 2 stage sampling process using the multinomial and Gaussian distribution. Make sure you reference the parameters in your description.

Solution:

  • First, draw a multinomial distribution according to $p_1, \dots, p_k$, call the output $X$
  • If $X = i$, draw a Gaussian random variable from $Normal(\mu_i, \sigma^2_i)$.

binary_rmixgauss2 <- function(n, mix_probs, mus, sds) {

}

binary_rmixgauss3 <- function(n, mix_probs, mus, sds) {
    rolls <- sample(c(TRUE, FALSE), n, replace=TRUE, prob=mix_probs)
    values <- vector('numeric', n)
    for(i in seq_along(rolls)){
        if(rolls[i]) {
            out <- rnorm(1, mus[1], sds[1])
        } else {
            out <- rnorm(1, mus[2], sds[2])
        }
        values[i] <- out
    }
    return(values)
}

# They don't have to catch EVERYTHING, but just have an idea of how to do this
check_rmixgauss_input <- function(n, mix_probs, mus, sds) {
    if(n < 0) {
        stop("n cannot be negative")
    }
    if(length(n) != 1) {
        stop("n should be length 1")
    }
    k <- length(mix_probs)
    if(length(sds) != k){
        stop("length of sds does not match mix_probs")
    }
    if(length(mus) != k){
        stop("length of mus does not match mix_probs")
    }
    inputs <- list(n, mix_probs, mus, sds)
    values <- vector('logical', length(inputs))
    for(input in inputs){
        if(class(input) != 'numeric') stop("all inputs must be numeric")
    }
}

Problm 4: Simulating data and fitting the wrong model!¶

A method to fit a model to data is to search for the parameters that maximize the log-likelihood of observing the sample, i.e. $$\theta_{MLE} = \arg\max_{\theta} \log p_{\theta}(data)$$ So if we assume the data follows a Normal distribution and the samples are independent from one another, we can rewrite this as: $$\theta_{MLE} = \arg\max \log \prod_i \frac{1}{\sigma\sqrt{2\pi}} \exp\left(-\frac{1}{2} (\frac{y_i - \mu}{\sigma})^2\right)$$
